# AI嵌入式（边缘视觉方向）工程师 - 岗位需求

## 需求名称

AI嵌入式（边缘视觉方向）工程师

## 需求内容

### 岗位职责

1. **视觉模型边缘部署**。将各类视觉模型部署到瑞芯微/高通等 SoC 或 NPU 板子上，打通从摄像头输入到推理输出的全链路。

2. **性能优化与模型压缩**。针对各业务场景的实时要求（帧率、延时、功耗），进行模型剪枝、蒸馏、量化（INT8/FP16 等）、结构重写、算子融合等优化，达到边缘侧可用的性能指标。

3. **推理引擎与算子优化**。熟练使用对应平台的推理引擎/SDK（如 RKNN-Toolkit2、RKNNToolkitLite2、Qualcomm SNPE/QNN、TensorRT、ONNXRuntime），进行算子适配、kernel 优化，分析并解决算子不支持、精度掉点、性能瓶颈等问题。

4. **系统集成与数据链路打通**。负责摄像头/传感器采集、图像预处理（Resize、Normalize、NV12/RGB 转换等）、推理后处理（NMS、跟踪、多目标管理）在板端的实现与调优，保证系统稳定性、实时性与资源占用可控。

5. **工具链与脚本开发**。编写和维护模型转换、量化、打包部署、自动测试等脚本和工具（Python/Shell），沉淀可复用的部署工具链和内部最佳实践。

6. **多场景边缘 AI 落地**。负责板端 AI 模块的技术方案设计、版本迭代与问题定位，支撑产品规模化交付。

7. **关注各类新形态硬件平台上的 AI 能力**，参与大模型/视觉模型在这些设备上的本地部署或端云协同方案的预研和 PoC 实现。

### 任职要求

1. **学历背景**：本科及以上学历，计算机、电子信息、自动化、通信工程、控制工程、机器人等相关专业。

2. **编程与工程能力**：熟练掌握 C 或 C++，具备良好的指针、内存管理与多线程编程基础；熟悉至少一种脚本语言，Python 优先，能独立编写模型转换、测试和运维脚本。

3. **嵌入式开发经验**：有嵌入式 Android/Linux 平台开发经验，了解 ARM 架构、交叉编译、驱动/系统移植的基本流程；至少在瑞芯微/高通/Jetson 等平台上做过实际项目（其中瑞芯微为必须项）。

4. **深度学习与模型部署基础**：熟悉 PyTorch，理解基础训练流程和模型导出（ONNX/TorchScript 等）；理解常见视觉任务（检测、分类、分割、关键点/姿态等）的基本网络结构和评估指标。

5. **模型压缩与加速经验**：对剪枝、量化、知识蒸馏、算子融合、流水线并行、异构加速（CPU/GPU/NPU）等概念有实战经验，至少主导/参与过 1–2 个模型压缩与性能优化项目。

6. **推理引擎与 NPU SDK**：熟悉 RKNN、SNPE/QNN、ONNX Runtime 等，能完成模型转换、调试与性能分析。

7. **调试与问题定位能力**：能在资源受限环境下使用 profiling 工具或自定义日志分析 CPU/GPU/NPU 利用率、内存、带宽等瓶颈，定位并解决崩溃、卡顿、内存泄漏、结果异常等问题。

8. **具备良好的沟通能力和文档能力**。
